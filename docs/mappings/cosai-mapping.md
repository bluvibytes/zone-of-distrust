> **Mapping Disclaimer:** This document maps the referenced framework to Zones of Distrust (ZoD) for architectural alignment purposes. ZoD is **complementary, not a replacement**, and this mapping does **not** claim certification, compliance, endorsement, or affiliation. Control alignment depends on correct implementation, configuration, and supporting security practices. "Mapped" indicates enforcement, detection/containment, or audit evidence support—not guaranteed prevention.

---



## Overview

The Coalition for Secure AI (CoSAI) is an OASIS Open Project with founding members including Anthropic, Amazon, Cisco, Google, IBM, Intel, Microsoft, NVIDIA, OpenAI, and PayPal. The CoSAI Risk Map (CoSAI-RM) provides an open framework for AI security risk identification.

**Relationship:** CoSAI-RM identifies industry-consensus *risks*. ZoD provides the *architectural controls* to address them.

## CoSAI-RM Risk Categories

### Category 1: Model Risks

**CoSAI Definition:** Risks inherent to the AI model itself, including training, architecture, and behavior.

| CoSAI Risk | ZoD Control | Layer |
|------------|-------------|-------|
| Training data poisoning | Model provenance, behavioral monitoring | L1, L6 |
| Model extraction | Infrastructure security (out of scope) | — |
| Adversarial inputs | Input screening, behavioral baseline | L2, L6 |
| Emergent capabilities | Semantic policy constraints | L4 |
| Model drift | Behavioral baseline monitoring | L6 |

**Key ZoD Property:** L6 behavioral monitoring detects model-level anomalies.

---

### Category 2: Data Risks

**CoSAI Definition:** Risks related to data used by, processed by, or generated by AI systems.

| CoSAI Risk | ZoD Control | Layer |
|------------|-------------|-------|
| Data leakage | Semantic policy egress control | L4 |
| Data poisoning | Input screening, memory audit | L2, L6 |
| Privacy violations | Data classification policies | L4 |
| Data integrity | Content hashing, audit trails | L4, Integrity |

**Key ZoD Control:** L4 semantic intent policy includes data classification and egress rules.

---

### Category 3: Infrastructure Risks

**CoSAI Definition:** Risks to the compute, storage, and network infrastructure supporting AI.

| CoSAI Risk | ZoD Control | Layer |
|------------|-------------|-------|
| Compute compromise | L1 infrastructure integrity (boundary) | L1 |
| Network attacks | Network isolation, inter-layer security | L3 |
| Supply chain compromise | Model provenance, component attestation | L1 |
| Configuration drift | GitOps policy management | L7 |

**Key ZoD Boundary:** L1 infrastructure integrity is a trust assumption—if L1 is fully compromised, architecture cannot defend.

---

### Category 4: Application Risks

**CoSAI Definition:** Risks in how AI is integrated into applications and workflows.

| CoSAI Risk | ZoD Control | Layer |
|------------|-------------|-------|
| Prompt injection | Pre-reasoning input screening | L2 |
| Output manipulation | Action validation, parameter binding | L4, L5 |
| Session hijacking | Token-based execution, replay prevention | L4 |
| API abuse | Rate limiting, behavioral monitoring | L6 |

**Key ZoD Control:** P1 (No Unmediated Execution)—all application actions require CA validation.

---

### Category 5: Operational Risks

**CoSAI Definition:** Risks in operating and maintaining AI systems in production.

| CoSAI Risk | ZoD Control | Layer |
|------------|-------------|-------|
| Monitoring gaps | Continuous behavioral monitoring | L6 |
| Incident response failures | Recovery protocols | L7 |
| Change management risks | GitOps, approval workflows | L7 |
| Alert fatigue | Cross-layer correlation, risk scoring | L6 |

**Key ZoD Properties:** P6 (Cross-Layer Correlation), P7 (Immutable Forensic Record)

---

### Category 6: Governance Risks

**CoSAI Definition:** Risks from inadequate oversight, policy, and accountability structures.

| CoSAI Risk | ZoD Control | Layer |
|------------|-------------|-------|
| Unclear accountability | Cryptographic attribution | P2 |
| Policy gaps | Comprehensive semantic policy | L4, L7 |
| Compliance failures | Regulatory mapping documentation | Docs |
| Oversight failures | Human governance, escalation | L7 |

**Key ZoD Property:** P11 (External Policy Authority)—humans define policy, agents execute.

---

### Category 7: Third-Party Risks

**CoSAI Definition:** Risks from external dependencies, vendors, and integrations.

| CoSAI Risk | ZoD Control | Layer |
|------------|-------------|-------|
| Vendor compromise | Supply chain verification | L1 |
| API dependency failures | Graceful degradation | P9 |
| Third-party model risks | Model provenance | L1 |
| Integration vulnerabilities | Inter-system screening | L2 |

**Key ZoD Control:** L1 model provenance extends to all third-party components.

---

### Category 8: Agentic Risks

**CoSAI Definition:** Risks specific to autonomous AI agents and multi-agent systems.

| CoSAI Risk | ZoD Control | Layer |
|------------|-------------|-------|
| Goal hijacking | Input screening, semantic policy | L2, L4 |
| Excessive autonomy | Risk-weighted escalation | L7 |
| Multi-agent compromise | Chain validation, inter-agent screening | L2, L4 |
| Memory poisoning | External memory audit | L6 |
| Tool misuse | Parameter binding, tool constraints | L4 |

**Key ZoD Insight:** ZoD was designed specifically for agentic risks—this is its core domain.

## Verification Checklist

**Model Risks (Category 1)**
- [ ] L1 model provenance verified
- [ ] L6 behavioral baseline established
- [ ] L4 semantic constraints on capabilities

**Data Risks (Category 2)**
- [ ] L4 data classification rules defined
- [ ] L4 egress controls operational
- [ ] L6 memory audit scheduled

**Infrastructure Risks (Category 3)**
- [ ] L1 infrastructure integrity verified
- [ ] L3 network isolation enforced
- [ ] L7 GitOps configuration management

**Application Risks (Category 4)**
- [ ] L2 input screening active
- [ ] L4 action validation operational
- [ ] P1 enforced—no unmediated execution

**Operational Risks (Category 5)**
- [ ] L6 continuous monitoring active
- [ ] P6 cross-layer correlation operational
- [ ] Recovery protocols documented and tested

**Governance Risks (Category 6)**
- [ ] P11 policy authority established
- [ ] L7 escalation workflows defined
- [ ] Compliance documentation complete

**Third-Party Risks (Category 7)**
- [ ] L1 third-party provenance verification
- [ ] P9 degradation modes for dependency failures

**Agentic Risks (Category 8)**
- [ ] L2 inter-agent screening enabled
- [ ] L4 chain validation operational
- [ ] L6 memory audit active
- [ ] L7 autonomy levels enforced
